## 🎯 Objective

The **Linux Labyrinth** challenge series focused on developing investigative skills within a Linux environment using command-line tools.

Participants were required to navigate the filesystem, locate hidden artifacts, interact with local services, and analyze binary data to recover hidden information.

These exercises simulate the types of exploratory actions analysts perform during:

* incident response investigations
* system compromise analysis
* forensic artifact discovery

The goal of the series was to demonstrate how **systematic command-line exploration** can uncover hidden data within complex environments.

To respect competition integrity:

* ❌ Flags are not published
* ❌ Protected answers are not disclosed
* ✔ Only investigation methodology is documented

## 🛠 Tools Used

The following command-line utilities were used across the Linux Labyrinth investigations.

| Tool      | Purpose                             |
| --------- | ----------------------------------- |
| `ls`      | Directory enumeration               |
| `pwd`     | Identify current working directory  |
| `cd`      | Navigate filesystem                 |
| `find`    | Search for files                    |
| `cat`     | Inspect file contents               |
| `strings` | Extract readable text from binaries |
| `nmap`    | Identify running network services   |
| `curl`    | Interact with HTTP services         |
| `xxd`     | Convert hex data to binary          |

These tools represent foundational techniques used during Linux investigations and system analysis.
