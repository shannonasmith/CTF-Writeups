## 🎯 Objective

Investigate historical versions of a website to identify information that is no longer publicly visible in the current version.

The goal of this challenge was to use **Open Source Intelligence (OSINT)** techniques to locate archived content and recover information that had been removed or altered over time.

This exercise focused on understanding how publicly available archives can preserve historical web artifacts.

---

## 🖥 Environment

- Web browser  
- Internet archive resources  
- OSINT investigation techniques  

---

## 🔍 Step 1 — Analyze the Target Website

The investigation began by examining the current version of the provided website.

Initial analysis focused on:

- reviewing visible content  
- identifying references to historical information  
- observing page structure and available resources  

The site itself did not immediately expose the required information.

This suggested the need to investigate **previous versions of the site**.

---

## 🔎 Step 2 — Investigate Historical Snapshots

Public web archives maintain historical records of websites at different points in time.

Using an archive service, the domain was searched to identify:

- previous snapshots of the website  
- historical page content  
- earlier versions of resources no longer available on the live site  

Archived versions allow investigators to view how a website appeared in the past.

---

## 🔄 Step 3 — Examine Older Versions of the Page

Several archived snapshots were reviewed to determine whether information existed in earlier versions that had since been removed.

Older versions of the site sometimes contained:

- different page content  
- additional resources  
- references that were no longer present in the current site  

Careful review of these snapshots revealed information that was no longer visible in the live version.

---

## 🔐 Step 4 — Identify Historical Artifact

Through analysis of archived versions of the website, previously visible information was located and examined.

This confirmed that the data had existed in an earlier version of the site and had later been removed or modified.

The archived snapshot preserved the historical content, allowing the investigation to recover the necessary information.

---

# 🧠 Methodology Framework Applied

1. Analyze the current website  
2. Identify potential missing or historical content  
3. Search for archived versions of the domain  
4. Review historical snapshots  
5. Compare older versions with the current site  
6. Recover information preserved in archived content  

---

# 🛡 Defensive Insight

Information removed from a website may still exist in publicly accessible archives.

Organizations should understand that:

- historical web content may persist indefinitely  
- sensitive data published publicly can remain accessible even after removal  
- proper data handling practices are critical before publishing information online  

Once information becomes public, it may be **difficult or impossible to completely erase**.
